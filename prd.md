# Product Requirements Document (PRD)
**Project Name:** ComShopX - All-in-One Computer Store Management System  
**Version:** 1.0.0  
**Date:** 6 June 2026

## 1. Introduction
ComShopX เป็นเว็บแอปพลิเคชันสำหรับบริหารจัดการร้านขายอุปกรณ์คอมพิวเตอร์และรับจัดสเปคแบบครบวงจร ระบบถูกออกแบบมาเพื่อลดความซับซ้อนในการทำงานของพนักงานหน้าร้านและเจ้าของร้าน โดยครอบคลุมตั้งแต่การขายหน้าร้าน (POS), การจัดการสต๊อกสินค้าแบบอิง Serial Number, การติดตามสถานะประกัน ไปจนถึงการออกเอกสารสำคัญทางธุรกิจ

## 2. Objectives
- สร้างระบบที่สามารถออกเอกสารธุรกิจได้ครบวงจร ได้แก่ ใบเสนอราคา, ใบส่งของ และใบเสร็จรับเงิน/ใบกำกับภาษี
- จัดการคลังสินค้าคอมพิวเตอร์ที่ต้องการความแม่นยำสูง โดยติดตามสินค้าเป็นรายชิ้นผ่าน Serial Number
- ลดเวลาการทำงานซ้ำด้วย workflow แปลงเอกสาร เช่น ใบเสนอราคา -> ใบส่งของ -> ใบเสร็จรับเงิน/ใบกำกับภาษี
- รองรับงานขายหน้าร้านด้วย POS ที่คำนวณส่วนลด, VAT 7%, การรับชำระเงิน และการตัดสต๊อกอัตโนมัติ
- เก็บประวัติการซื้อของลูกค้าและข้อมูล Serial Number เพื่อให้ค้นหาประวัติรับประกันและเคลมได้รวดเร็ว

## 3. Target Users
- **Admin/Owner:** ดูภาพรวมรายได้, สต๊อกคงเหลือ, เอกสารสำคัญ, กำไรขั้นต้น และจัดการข้อมูลหลัก
- **Cashier/Sale:** สร้างใบเสนอราคา, จัดสเปคคอมพิวเตอร์, ขายหน้าร้าน, เลือก Serial Number ตอนขาย และออกใบเสร็จ
- **Inventory Staff:** รับสินค้าเข้า, บันทึก Serial Number, ตรวจนับสต๊อก, โอนย้ายสถานะสินค้า และดูรายการใกล้หมด
- **Accounting Staff:** ตรวจสอบเอกสาร, เลขที่เอกสาร, VAT, ข้อมูลลูกค้า และส่งออก PDF สำหรับงานบัญชี

## 4. Core Features

### 4.1 Document Management
- **Quotation (ใบเสนอราคา):** สร้างใบเสนอราคาพร้อมวันหมดอายุ, ส่วนลด, VAT 7%, หมายเหตุ และ Export PDF
- **Delivery Note (ใบส่งของ):** สร้างใบส่งของจากใบเสนอราคาได้ทันที พร้อมระบุสินค้าที่ส่งมอบจริง
- **Receipt / Tax Invoice (ใบเสร็จรับเงิน/ใบกำกับภาษี):** ออกเอกสารเมื่อรับชำระเงิน รองรับเลขประจำตัวผู้เสียภาษีและเลขที่เอกสารอัตโนมัติ
- **Document Conversion:** แปลงสถานะเอกสารและอ้างอิงเอกสารเดิมได้ เช่น QUO-202606-001 -> DN-202606-001 -> REC-202606-001
- **Document Status:** รองรับ DRAFT, ISSUED, PAID, VOID และ CANCELLED

### 4.2 Inventory Management
- จัดการสินค้า, หมวดหมู่, SKU, Barcode, ราคาขาย, ราคาทุน และจุดสั่งซื้อขั้นต่ำ
- **Serial Number Tracking:** สินค้าที่กำหนด `require_serial` ต้องเลือก Serial Number ตอนขายหรือส่งของเสมอ
- สถานะ Serial Number ได้แก่ IN_STOCK, RESERVED, SOLD, CLAIMED, RETURNED และ LOST
- รับสินค้าเข้าแบบระบุ Serial Number ทีละรายการหรือ bulk import
- แจ้งเตือนสินค้าใกล้หมดสต๊อกและ Serial Number ที่มีสถานะผิดปกติ
- เก็บประวัติการเคลื่อนไหวของสินค้าเพื่อ audit ย้อนหลัง

### 4.3 Customer Management
- บันทึกชื่อลูกค้า, ที่อยู่, เบอร์โทร, อีเมล และเลขประจำตัวผู้เสียภาษี
- แยกประเภทลูกค้าเป็นบุคคลธรรมดาและนิติบุคคล
- ดูประวัติเอกสาร, การซื้อ, Serial Number ที่เคยซื้อ และงานเคลมย้อนหลัง

### 4.4 Point of Sale (POS)
- หน้าจอขายสินค้าที่รองรับ Barcode/QR Code scanner
- ตะกร้าสินค้า, ส่วนลดรายบรรทัด, ส่วนลดท้ายบิล และ VAT 7%
- เลือก Serial Number สำหรับสินค้าที่ต้องติดตามรายชิ้น
- รองรับการชำระเงินสด, โอน, บัตรเครดิต และหลายช่องทางในบิลเดียว
- ออกใบเสร็จรับเงิน/ใบกำกับภาษีหลังรับชำระเงิน

### 4.5 Reporting
- Dashboard ยอดขายรายวัน, รายเดือน และสินค้าขายดี
- รายงานสินค้าใกล้หมด, สินค้าคงคลังตาม Serial Number และเอกสารค้างชำระ
- Export CSV/PDF สำหรับงานบัญชีและตรวจนับสต๊อก

## 5. Non-Functional Requirements
- **Performance:** หน้าเอกสารและ PDF preview ต้องโหลดภายใน 2 วินาทีสำหรับเอกสารขนาดทั่วไป
- **Security:** มี Authentication, Role-based Access Control (RBAC), audit log และการป้องกันการแก้ไขเอกสารที่ออกแล้วโดยไม่มีสิทธิ์
- **Usability:** UI รองรับ desktop และ tablet สำหรับพนักงานขายและสต๊อก
- **Reliability:** การขายต้องทำแบบ transaction เพื่อให้การออกเอกสาร, การตัดสต๊อก และการเปลี่ยนสถานะ Serial Number สำเร็จหรือ rollback พร้อมกัน
- **Data Integrity:** Serial Number ต้อง unique และไม่สามารถขายซ้ำได้
- **Localization:** รองรับภาษาไทย, รูปแบบเลขที่เอกสารไทย และ VAT 7%

## 6. Success Metrics
- ลดเวลาการออกใบเสนอราคาและใบเสร็จเหลือไม่เกิน 2 นาทีต่อรายการขายทั่วไป
- ลดปัญหาขายสินค้า Serial Number ซ้ำเป็น 0 เคส
- เจ้าของร้านสามารถตรวจสอบสต๊อกและยอดขายรายวันได้จาก Dashboard เดียว
- เอกสารธุรกิจมีเลขที่ถูกต้องและค้นหาย้อนหลังได้ครบถ้วน
